# 出题喵喵 QuizMiao — 产品需求文档 (PRD)

> 版本：v1.3 | 最后更新：2026-07-17

---

## 1. 产品概述

### 1.1 产品定位

出题喵喵是一款 **AI 驱动的练习题自动生成工具**。用户粘贴任意文本或网页链接，AI 自动从中提取知识点并生成四选一选择题，支持在线练习、错题回顾和分享给好友。

核心价值：**将任何学习资料在 30 秒内转化为可自测的练习题**。

### 1.2 目标用户

| 用户画像 | 典型场景 |
|----------|----------|
| 在校学生 | 将课堂笔记/教材内容转为练习题，考前自测 |
| 自学爱好者 | 阅读文章/教程后生成题目，检验理解 |
| 教师/培训师 | 根据讲义快速生成课堂测验 |
| 知识分享者 | 生成题目后通过链接/二维码分享给社群好友一起做题 |

### 1.3 产品形态

| 平台 | 地址/入口 | 技术栈 |
|------|-----------|--------|
| Web 版 | `https://philcui07.github.io/quizmiao/` | 静态前端 (GitHub Pages) + 后端代理 (Vercel Serverless) |
| 微信小程序 | 「出题喵喵」 | 微信小程序原生框架 + 云函数代理 |

## 2. 功能范围

### 2.1 功能模块总览

```
出题喵喵 QuizMiao
├── 首页 (index)
│   ├── 文本输入 → AI 出题
│   └── 链接抓取 → AI 出题
├── 确认页 (confirm)
│   ├── 题目列表预览（流式加载）
│   ├── 删除单题
│   ├── 重新生成
│   ├── 分享弹窗（链接 / 二维码）
│   └── 开始练习
├── 练习页 (practice)
│   ├── 逐题作答（四选一）
│   ├── 即时反馈（正确 / 错误 + 解析）
│   └── 得分追踪
└── 结果页 (result)
    ├── 得分统计（分数 + 正确率）
    ├── 错题回顾（含正确答案和解析）
    ├── 再做一次 / 返回题目列表
    └── 分享（链接 / 二维码）
```

### 2.2 功能详解

#### 2.2.1 内容输入（首页）

| 功能点 | 描述 |
|--------|------|
| **文本 Tab** | 用户粘贴文本内容（笔记、文章、单词表等），最少 20 字符 |
| **链接 Tab** | 用户输入 1~N 个网页链接，后端抓取 HTML 并提取纯文本 |
| **出题数量** | Web 版：5/10/20/30/40/50 题可选，默认 10；小程序：5/10/20/30 题可选，默认 10 |
| **生成按钮** | 点击后跳转确认页，开始流式出题 |

#### 2.2.2 AI 出题（后端代理）

| 功能点 | 描述 |
|--------|------|
| **AI 模型** | DeepSeek v4 Flash（`deepseek-v4-flash`） |
| **传输模式** | SSE 流式传输（Web 直连 + 小程序云函数内部消费流再返回） |
| **题目格式** | 四选一选择题 `{cat, q, options[4], answer(0-3), exp}` |
| **质量保证** | 选项长度均匀性校验、答案均匀分布、题目前后验证（可选） |
| **超时** | 60 秒 |
| **降级策略** | SSE 不可用时自动回退到非流式 HTTP 请求 |

#### 2.2.3 题目确认页

| 功能点 | 描述 |
|--------|------|
| **流式进度** | 标题区域显示 "AI 正在出题..." + 三圆点动画；生成后显示 "✨ 已生成 X/Y 题" + 进度条 |
| **题目卡片** | 逐条滑入动画，显示：分类标签 / 题干 / 四个选项 / 正确答案 |
| **标签统计** | 按知识点分类汇总（如 "英语语法 ×3"、"数学公式 ×2"） |
| **删除题目** | 每题右上角 × 按钮，删除后重新渲染列表 |
| **重新生成** | 返回首页重新输入 |
| **分享** | 弹出选择面板：分享链接 / 分享二维码 |
| **开始练习** | 打乱题目顺序后进入练习页，题目不足时跳转首页 |

#### 2.2.4 练习页

| 功能点 | 描述 |
|--------|------|
| **进度条** | 顶部显示 "第 X/Y 题" + 得分 |
| **答题** | 点击 A/B/C/D 选项，正确答案高亮绿色，错误答案高亮红色 |
| **解析** | 答对/答错后立即显示题目解析 |
| **下一题** | 未作答时按钮禁用，作答后解锁 |
| **自动跳转** | 做完所有题目后自动跳转结果页 |

#### 2.2.5 结果页

| 功能点 | 描述 |
|--------|------|
| **得分** | 大号数字显示 X/Y，正确率百分比 |
| **评语** | 满分 "太棒了，全对！🎉"；80%+ "表现不错！"；其余 "继续加油！" |
| **错题回顾** | 每题显示题干、用户答案（红色）、正确答案（绿色）、解析 |
| **操作按钮** | 再做一次（重新打乱）/ 题目列表 / 返回首页 |

#### 2.2.6 分享功能

| 功能点 | 描述 |
|--------|------|
| **分享触发** | 确认页底部「分享」按钮 → 弹出选择面板 |
| **分享链接** | 后端 POST `/share` 存储题目数据 → 返回 6 位短 ID → 拼成 `#s=ABC123` 链接 → 自动复制到剪贴板 |
| **分享二维码** | 使用短链接调用 `api.qrserver.com` 生成二维码图片 → 复制到剪贴板 |
| **过期策略** | 分享数据 **1 小时后自动过期**，后端定时清理过期数据 |
| **打开分享** | 接收者打开链接 → 前端检测 `#s=` → GET `/share?id=xxx` 获取题目 → 跳转确认页 |

## 3. 分享系统设计

### 3.1 设计目标

- **简单**：分享者一键生成短链接，接收者一键打开做題
- **轻量**：不使用数据库，依赖 Vercel Serverless 内存存储（免费）
- **可控**：分享数据有明确的有效期，过期后友好提示
- **隐私**：只有主动分享的内容才会被他人访问

### 3.2 数据流

```
分享者                          后端 Vercel                    接收者
  │                                │                              │
  │  POST /share {questions}       │                              │
  │ ─────────────────────────────> │                              │
  │                                │  存 Map<id, {questions,      │
  │                                │          expiresAt}>         │
  │  返回 {ok:true, id:"ABC123"}  │                              │
  │ <───────────────────────────── │                              │
  │                                │                              │
  │  拼成 URL: #s=ABC123           │                              │
  │  复制到剪贴板                   │                              │
  │  ...粘贴到微信/聊天...          │                              │
  │                                │                              │
  │                                │  GET /share?id=ABC123         │
  │                                │ <─────────────────────────── │
  │                                │                              │
  │                                │  检查 expiresAt > now?        │
  │                                │  是 → 返回 questions           │
  │                                │  否 → 返回 404 +              │
  │                                │   "分享已过期(有效时长1小时)"  │
  │                                │ ───────────────────────────> │
  │                                │                              │
  │                                │  接收者看到题目列表            │
```

### 3.3 过期机制

| 项目 | 设定 |
|------|------|
| **TTL** | 1 小时（`60 * 60 * 1000` ms） |
| **存储** | Vercel Serverless 实例内存 `Map`，实例回收则数据丢失 |
| **清理** | 每次 POST/GET 调用 `cleanupShares()` 清理过期条目 |
| **过期提示** | "分享已过期（有效时长1小时），请让分享者重新生成" |

### 3.4 兼容性

- **新版短链接** `#s=ABC123`：后端短 ID 映射
- **旧版长链接** `#q=<base64>`：题目数据直接编码在 URL 中，无过期（向后兼容，仅 Web 版支持）

### 4.2 技术选型

| 层 | 选型 | 原因 |
|----|------|------|
| **Web 前端** | 原生 HTML + CSS + JS（无框架） | 零依赖、极小体积、GitHub Pages 直接托管 |
| **Web 后端** | Vercel Serverless (Node.js) | 免费、全球 CDN、与 GitHub 自动集成部署 |
| **小程序** | 微信原生框架 | 与微信生态深度整合，支持云开发 |
| **小程序后端** | 微信云函数 + 云数据库 | 免服务器运维、内网直连、低延迟 |
| **AI 模型** | DeepSeek v4 Flash | 高性价比、中文能力强、支持流式输出 |

### 4.3 后端 API 路由

| 方法 | 路径 | 功能 | 说明 |
|------|------|------|------|
| GET | `/?url=xxx` | 网页抓取 | 抓取 HTML 提取纯文本（最大 50000 字） |
| POST | `/llm` | AI 出题（非流式） | 一键返回全部题目 |
| POST | `/llm-stream` | AI 出题（SSE 流式） | 逐题推送，降低等待焦虑 |
| POST | `/verify` | 题目验证 | AI 二次验证答案正确性 |
| POST | `/share` | 存储分享数据 | 存到内存 Map，返回 6 位短 ID |
| GET | `/share?id=xxx` | 获取分享数据 | 从内存 Map 读取，过期返回 404 |
| GET | `/s/:encoded` | SEO 分享页 | 爬虫返回 OG meta 标签，普通请求 302 跳转 |

## 5. 数据模型

### 5.1 题目对象 (Question)

```typescript
interface Question {
  cat: string;      // 知识点分类，如 "英语语法"、"数学公式"
  q: string;        // 题干
  options: string[4]; // 四个选项（A/B/C/D）
  answer: number;   // 正确答案下标 0-3
  exp: string;      // 解析说明
}
```

### 5.2 练习状态 (Store)

```typescript
interface Store {
  questions: Question[];  // 当前题库
  pool: Question[];       // 打乱后的练习池
  idx: number;            // 当前题号
  score: number;          // 得分
  wrong: WrongItem[];     // 错题记录
}

interface WrongItem {
  q: string;
  cat: string;
  picked: string;   // 用户选择的答案文本
  correct: string;  // 正确答案文本
  exp: string;      // 解析
}
```

### 5.3 分享存储 (ShareStore)

```typescript
// 存储结构（Vercel 内存）
Map<string, {
  questions: Question[];
  expiresAt: number; // Date.now() + 3600000 (1小时)
}>

// Key: 6位大写字母+数字短ID，如 "ABC123"
// 生成：Math.random().toString(36).substring(2,8).toUpperCase()
```

### 5.4 小程序云数据库

| 集合 | 用途 | 关键字段 |
|------|------|----------|
| `shared_quizzes` | 小程序分享数据存储 | `_id`, `questions`, `createdAt` |

## 6. 用户流程

### 6.1 出题 → 练习完整流程

```
[首页] → 输入内容 → 点击"生成练习题"
    │
    ▼
[确认页] ← 流式加载题目
    │         ├─ 看到进度条和逐题出现动画
    │         ├─ 可删除不满意的题目
    │         ├─ 可分享给好友
    │         └─ 点击"开始练习"
    ▼
[练习页] ← 逐题作答
    │         ├─ 选择答案 → 即时反馈
    │         ├─ 查看解析
    │         └─ 点击"下一题"（循环至最后一题）
    ▼
[结果页] ← 得分 + 错题回顾
    │         ├─ "再做一次" → 返回练习页（重新打乱）
    │         ├─ "题目列表" → 返回确认页
    │         └─ "返回首页" → 返回首页
```

### 6.2 分享 → 接收流程

```
[分享者确认页] → 点击分享按钮
    │
    ▼
[分享弹窗]
    ├─ "分享链接" → POST /share → 短链接 → 复制到剪贴板
    └─ "分享二维码" → POST /share → 短链接 → 生成二维码 → 复制图片
    │
    ▼
[微信/QQ/聊天窗口] ← 粘贴链接或二维码图片
    │
    ▼
[接收者打开链接]
    │
    ▼
[确认页] ← 加载分享内容
    │         ├─ 成功 → 显示全部题目 → 可直接开始练习
    │         └─ 过期 → 提示"分享已过期（有效时长1小时），请让分享者重新生成"
```

## 7. 非功能需求

### 7.1 性能

| 指标 | 目标 | 备注 |
|------|------|------|
| 首屏加载 | < 2s | 纯静态，无外部依赖 |
| AI 出题首题时间 | < 5s | SSE 流式首题 |
| 全部题目生成 | < 30s | 50 题场景 |
| 页面切换 | < 100ms | 纯 CSS 显隐切换 |
| 网页抓取超时 | 15s | 防止加载慢的网站阻塞 |

### 7.2 兼容性

| 平台 | 最低版本 |
|------|----------|
| Web (PC) | Chrome 80+, Firefox 75+, Safari 13+, Edge 80+ |
| Web (Mobile) | iOS Safari 13+, Android Chrome 80+, 微信内置浏览器 |
| 微信小程序 | 基础库 2.0+ |

### 7.3 安全

| 策略 | 说明 |
|------|------|
| **API Key 保护** | DeepSeek API Key 仅在 Vercel 环境变量中，前端不可见 |
| **内容安全** | 分享数据 1 小时后自动过期，不留存 |
| **HTTPS** | 所有 API 通信使用 HTTPS |
| **CORS** | 后端开放 `Access-Control-Allow-Origin: *`（无敏感用户数据） |

### 7.4 可维护性

| 策略 | 说明 |
|------|------|
| **Web/小程序代码一致性** | 两套前端代码逻辑尽量同步，差异仅在于运行环境 API |
| **无框架依赖** | Web 版零 npm 依赖，小程序仅依赖微信 SDK |
| **Git + Vercel 自动部署** | 推送即部署，无需手动操作 |
| **版本号管理** | HTML 中 JS/CSS 引用带版本号参数（`?v=13`），防止缓存 |
