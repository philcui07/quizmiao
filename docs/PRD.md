# 拾知猫 QuizMiao 产品需求文档

> 产品版本：v1.1.0
> 文档状态：Web 开发版
> 更新日期：2026-07-23

## 1. 产品定位

拾知猫是一款 AI 驱动的练习题生成工具。用户输入文本或网页链接，系统提取内容并生成四选一练习题，支持在线答题、错题回顾、账号历史和好友分享。

核心价值：把学习资料快速转化为可以反复练习、追踪成绩和分享的题集。

## 2. 版本范围

### 2.1 v1.0.1

- 状态：冻结，不再改动。
- Web 后端：Vercel Serverless。
- 核心能力：文本/链接出题、练习、结果、临时分享。

### 2.2 v1.1.0

- 当前开发顺序：先完成并验收 Web，再开发 Miniapp。
- 后端统一迁移至腾讯云 CloudBase。
- Web 登录：优先接入运营商一键认证；未配置或失败时回退为手动手机号。
- Miniapp 登录：微信 `getPhoneNumber` 能力获取实名手机号，后续实现。
- 新增账号昵称、题集历史、多次练习成绩、错题集、分享命名和好友答题记录。
- 分享链接有效期为 24 小时；分享答题记录在分享人账号中继续保留。

## 3. 用户与账号

### 3.1 Web 登录

普通 Chrome、Safari、Edge、Firefox 及微信内置浏览器没有标准 API 可以直接读取当前 SIM 手机号。WebOTP 只能在用户授权后读取短信中的一次性验证码，不能返回手机号，因此本版本不使用 WebOTP 或短信验证码登录。

Web 登录分为两条路径：

1. 页面先建立由 CloudBase Web SDK 持久化的匿名安全身份，作为设备数据权限主键。
2. 已配置运营商认证产品时，前端 provider adapter 获取一次性 token。
3. token 发送给 phone-auth 云函数，由服务端向运营商/聚合认证服务校验；仅校验成功后写入 phone_verified=true。
4. provider 未配置、当前网络不支持或认证失败时，展开手动手机号输入。
5. 手动手机号不做真实性校验，保存为当前设备身份下的资料字段，标记 phone_verified=false。
6. 历史、分享和账号查询始终使用 CloudBase 服务端认证上下文，不接受手机号或客户端 user ID 作为授权依据。

手动填写同一个手机号不会跨设备找回数据。只有运营商 token 服务端校验成功后，才能通过 HMAC 号码绑定映射到 canonical owner，并归并跨设备数据。前端只显示脱敏手机号。

### 3.2 Miniapp 登录

Miniapp v1.1.0 使用微信小程序手机号快速验证组件。前端只上传微信返回的动态凭证，手机号解密与账号绑定必须在 CloudBase 云函数中完成，不在小程序端保存会话密钥。

### 3.3 昵称

- 登录用户可以设置 1-20 个字符的昵称。
- 昵称保存在账号资料集合中；手动手机号账号仅当前设备可读取，通过运营商认证归并后可跨设备读取。
- 未设置昵称时，Web 显示脱敏手机号。
- 被分享人填写的答题昵称与账号昵称相互独立。

## 4. 出题与练习历史

### 4.1 题集记录

登录用户每完成一次 AI 出题，确认页中的题目会形成一条题集记录。用户在确认页删除题目时，同步更新该题集，不重复创建记录。

题集列表展示：

- 题集名称；
- 题目数量；
- 已练习次数；
- 最近一次成绩和错题数量；
- 创建时间。

### 4.2 多次练习

同一题集可以反复练习。每次进入练习都会创建唯一轮次 ID，完成后在该题集下追加一条练习记录。

每次练习记录包含：

- 得分、总题数、正确率；
- 完成时间；
- 错题题干；
- 用户答案、正确答案和解析。

重复渲染结果页不能重复写入同一练习轮次。

## 5. 分享

### 5.1 创建分享

1. 用户在题目确认页点击分享。
2. 弹窗允许输入最多 50 个字符的分享名称。
3. CloudBase 保存题目、名称、创建人和过期时间。
4. 返回随机分享 ID，前端生成 `#s=<id>` 链接或二维码。
5. 分享链接自创建起 24 小时内可打开。

未登录用户仍可创建临时分享，但好友答题结果不会归入账号。登录用户创建的分享进入“我的分享”。

### 5.2 被分享人昵称

- 首次打开任意新分享链接时弹出昵称输入框。
- 昵称非必填，可以跳过并以“匿名用户”记录。
- 填写后保存在本机浏览器 `localStorage`。
- 同一设备再次打开任何人的拾知猫分享链接时自动沿用，不重复弹窗。

### 5.3 好友答题记录

通过分享链接完成练习后，系统把成绩和错题集写入该分享下。分享人可以在账号的“我的分享”中查看：

- 分享名称、题目数量和有效状态；
- 收到的答题次数；
- 每位答题人的昵称或“匿名用户”；
- 每次成绩、正确率、时间和错题集。

界面结构与本人练习记录保持一致。分享链接过期后不能再打开或提交新成绩，但分享人仍可查看已产生的答题记录。

## 6. Web 信息架构

```text
首页
├── 文本出题
├── 链接出题
├── 登录 / 账号菜单
└── 历史记录
    ├── 我的题集
    │   └── 题集详情
    │       ├── 多次练习记录
    │       ├── 每次成绩与错题集
    │       └── 题目列表
    └── 我的分享
        └── 分享详情
            ├── 好友答题记录
            ├── 每次成绩与错题集
            └── 题目列表
```

## 7. CloudBase 架构

### 7.1 云函数

| 云函数 | 职责 |
|---|---|
| `quiz-generate` | 调用 DeepSeek 生成和验证题目 |
| `page-fetch` | 安全抓取公开网页正文 |
| `profile-manage` | 读取和更新设备账号资料 |
| `phone-auth` | 服务端校验运营商 token、绑定认证号码并归并 canonical owner |
| `history-manage` | 题集与练习轮次管理 |
| `share-manage` | 分享创建、读取和账号列表 |
| `share-result` | 好友答题写入和分享人查询 |

### 7.2 数据集合

| 集合 | 主要数据 |
|---|---|
| `users` | CloudBase 设备身份、canonical owner、昵称、手机号资料及认证状态 |
| `phone_bindings` | 认证号码 HMAC 与 canonical owner 映射 |
| `quiz_history` | 一次出题形成的题集、练习次数、最近成绩 |
| `quiz_attempts` | 题集下每次练习的成绩和错题集 |
| `shares` | 分享名称、题目、创建人、24 小时过期时间 |
| `share_results` | 分享下每次好友答题成绩和错题集 |

所有集合禁止 Web 客户端直接读写，只能通过云函数访问。云函数从 CloudBase 服务端上下文读取用户 ID，不接受前端传入的用户 ID 作为授权依据。

### 7.3 环境变量

- `DEEPSEEK_API_KEY`：仅配置在 `quiz-generate` 云函数中。
- `DEEPSEEK_MODEL`：默认 `deepseek-v4-flash`。
- `PHONE_AUTH_PROVIDER`：服务端认证 adapter 名称；未接入时保持 `disabled`。
- `PHONE_AUTH_VERIFY_URL`：供应商或自建认证网关的 HTTPS token 校验地址。
- `PHONE_AUTH_APP_ID`、`PHONE_AUTH_VERIFY_SECRET`：认证项目标识及服务端密钥。
- `PHONE_HASH_SECRET`：至少 32 字符，仅用于服务端 HMAC 号码映射。
- CloudBase 环境 ID 不是密钥，可配置在 Web 前端。

## 8. 非功能需求

### 8.1 安全

- 普通 Web 不声明能够通过浏览器直接读取手机号。
- 手动手机号不得用于查找 owner、切换账号或授权历史数据。
- 运营商 token 必须在服务端校验，前端返回的手机号或认证标志一律不可信。
- 手机号仅写入关闭客户端权限的账号资料集合，不记录明文日志；号码绑定使用服务端 HMAC。
- 账号数据必须按服务端登录身份隔离。
- 网页抓取仅允许 HTTP/HTTPS 标准端口，禁止访问本机、内网和链路本地地址。
- DeepSeek API Key 不进入 Git、不返回前端。
- 分享和练习写入均做字段长度、数组数量和分数范围校验。

### 8.2 兼容性

- Web：当前主流 Chrome、Edge、Firefox、Safari 及微信内置浏览器。
- 响应式范围：320px 手机宽度至桌面浏览器。
- Miniapp：以开发时微信基础库稳定版本为准。

### 8.3 数据与费用

- 分享有效期 24 小时。
- 过期分享仍保留账号侧历史结果；上线前应在 CloudBase 设置数据保留和定期清理策略。
- AI、运营商认证和云函数均需设置日/月费用告警。
- 出题数量由服务端限制在 5-50 题，输入内容限制为 50,000 字符。

## 9. v1.1.0 验收标准

- 普通 Web 页面不包含短信验证码 UI 或 CloudBase 短信调用。
- provider 已配置时优先展示一键登录，token 只在服务端校验；不可用或失败时显示手动手机号回退。
- 手动输入的手机号标记为未验证，换设备输入相同号码不能读取原设备历史。
- 未登录点击历史入口会打开登录弹窗，登录成功后自动进入历史页。
- 运营商认证的同一号码可映射到 canonical owner，支持跨设备读取昵称和历史。
- 登录用户生成题目后，“我的题集”只新增一个题集。
- 同一题集练习两次后，详情显示两条独立成绩和各自错题集。
- 分享弹窗支持命名，链接在 24 小时后拒绝打开。
- 首次打开分享可填写或跳过昵称；填写后同机其他分享自动沿用。
- 登录分享人可以查看好友答题成绩和错题集。
- 客户端伪造手机号、认证状态或用户 ID 不能读取其他账号数据。
- Web 不再依赖 Vercel 处理业务请求。
